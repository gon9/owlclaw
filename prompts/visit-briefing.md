# 訪問先ブリーフィング生成

あなたは私のパーソナルアシスタントです。
明日の訪問先について事前情報をリサーチし、ブリーフィングを作成してください。

## 入力データ

`tmp/visit-briefing/` ディレクトリに明日のカレンダー予定 (Markdown) が保存されています。

## タスク

各イベントについて以下を実施してください:

1. **企業・組織の調査**: 参加者のメールドメインから企業名を特定し、WebSearch/WebFetch で情報収集
   - 事業内容・主力サービス・最新ニュース（直近 3 ヶ月）
   - 従業員規模・設立年・上場有無
   - 担当者の LinkedIn・SNS 情報（公開情報のみ）

2. **アジェンダの推定**: イベントタイトル・説明文から会議の目的を推察

3. **想定質問リスト**: 相手の立場から考えられる質問を 3〜5 個列挙

4. **出力ファイルの作成**:
   - Slack 通知: `tmp/visit-briefing/slack.txt`
   - Obsidian メモ: `tmp/visit-briefing/obsidian.md`

## 出力フォーマット

### slack.txt

```
📋 明日の訪問先ブリーフィング {日付}

【{イベントタイトル} @ {時刻}】
🏢 {企業名}: {事業内容1行}
📰 最新情報: {直近ニュース1件}
❓ 想定質問: {質問1} / {質問2}
📎 詳細 → Obsidian: Briefings/{ファイル名}
```

### obsidian.md

```markdown
# {企業名} 訪問ブリーフィング — {日付}

## 会議情報
- タイトル: {イベントタイトル}
- 日時: {日時}
- 場所: {場所}
- 参加者: {参加者リスト（ドメインのみ）}

## 企業情報
{調査結果を箇条書きで}

## 最新ニュース
{直近3ヶ月のニュースを3件程度、出典URL付き}

## 想定質問
1. {質問1}
2. {質問2}
3. {質問3}

## メモ欄
（会議後に記録）
```

## 注意事項

- 予定が 0 件の場合は「明日の外部訪問予定はありません」と slack.txt に書いてください
- 個人情報（フルメールアドレス・電話番号）は出力しないこと
- 情報源 URL を必ず記載すること
- 確認できなかった情報は「不明」と明記する
