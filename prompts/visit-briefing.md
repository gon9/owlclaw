# 明日の外出予定ブリーフィング生成

あなたは私のパーソナルアシスタントです。
明日の物理的な外出予定について事前情報をまとめ、ブリーフィングを作成してください。

## 入力データ

`tmp/visit-briefing/` ディレクトリに明日のカレンダー予定 (Markdown) が保存されています。

## イベントの振り分け

まず各イベントを **参加者の有無** で2種類に振り分けてください:

- **A. 外部参加者あり**（イベントに参加者メールアドレスが記載されている）
  → 商談・訪問とみなし、下記「A の手順」でフルブリーフィングを作成
- **B. 参加者なし**（自分だけの個人外出予定。例: 食事・飲み会・通院・美容院・イベント参加）
  → 会社調査は不要。「B の手順」で移動情報と一言メモのみ作成

## A の手順（外部参加者ありのイベント）

1. **企業・組織の調査**: 参加者のメールドメインから企業名を特定し、WebSearch/WebFetch で情報収集
   - 事業内容・主力サービス・最新ニュース（直近 3 ヶ月）
   - 従業員規模・設立年・上場有無
   - 担当者の LinkedIn・SNS 情報（公開情報のみ）

2. **アジェンダの推定**: イベントタイトル・説明文から会議の目的を推察

3. **想定質問リスト**: 相手の立場から考えられる質問を 3〜5 個列挙

## B の手順（参加者なしの個人外出）

1. **移動情報**: 場所が記載されていれば WebSearch/WebFetch で最寄り駅・所要時間の目安を調べる
   - 出発地: `profile.yaml` の `home_address` を参照（未設定なら省略）
2. **一言メモ**: イベント種別（飲食・通院・美容院・イベント等）を推定し、役立つ一言（予約確認・持ち物・店舗電話番号など、WebSearch で確認できたもののみ）を添える
3. 会社調査・想定質問は **不要**

## 出力ファイルの作成

- Slack 通知: `tmp/visit-briefing/slack.txt`
- Obsidian メモ: `tmp/visit-briefing/obsidian.md`（A のイベントがある場合のみ詳細を記載。B のみの日は移動情報程度で可）

## 出力フォーマット

### slack.txt

```
📋 明日の外出予定ブリーフィング {日付}

# A. 外部参加者ありのイベント
【{イベントタイトル} @ {時刻}】
🏢 {企業名}: {事業内容1行}
📰 最新情報: {直近ニュース1件}
❓ 想定質問: {質問1} / {質問2}
📎 詳細 → Obsidian: Briefings/{ファイル名}

# B. 個人外出予定
【{イベントタイトル} @ {時刻}】
📍 {場所} / 🚉 {最寄り駅・所要時間の目安}
📝 {一言メモ（あれば）}
```

### obsidian.md

```markdown
# {企業名} 訪問ブリーフィング — {日付}

## 会議情報
- タイトル: {イベントタイトル}
- 日時: {日時}
- 場所: {場所}
- 参加者: {参加者リスト（ドメインのみ）}

## 企業情報
{調査結果を箇条書きで}

## 最新ニュース
{直近3ヶ月のニュースを3件程度、出典URL付き}

## 想定質問
1. {質問1}
2. {質問2}
3. {質問3}

## メモ欄
（会議後に記録）
```

## 注意事項

- 予定が 0 件の場合は「明日の外出予定はありません」と slack.txt に書いてください
- A のイベントが無く B のみの日も、B の個人外出は必ず通知してください（「予定なし」にしない）
- 個人情報（フルメールアドレス・電話番号）は出力しないこと
- 情報源 URL を必ず記載すること
- 確認できなかった情報は「不明」と明記する
