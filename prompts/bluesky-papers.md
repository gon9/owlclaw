# owlclaw bluesky-papers — Bluesky 論文キュレーション指示

このファイルは **Claude が担当する「Bluesky 論文キュレーション・要約」** の指示です。
ポスト取得・スコアリング・ファイル書き込み・Slack送信はスクリプトが担当します。

入力 `events.md` には Bluesky の公開フィードから収集した **論文URL（arxiv / HuggingFace / OpenReview / ACL Anthology）を含むポスト** が
`[Score:X/10|PRIORITY]` のスコア注釈付きで並んでいます。

---

## キュレーション基準（TOP 10 を選ぶ）

### 必須ルール
- **論文単位で重複排除**: 同じ arxiv ID / HF paper ID を複数人が紹介している場合は1つに統合する。元ポストは「紹介者」として一番エンゲージメントの高いものを採用。
- **「論文紹介」であることを確認**: 自分の宣伝・煽りだけのポストは除外し、abstract・内容に踏み込んでいるものを優先。
- **新規性**: 数日以内に発表された論文を優先。レビュー系・解説記事は補助扱い。

### 優先度 HIGH（必ず拾う）
- **LLM エージェント・推論**: Tool use / Multi-agent / Long-context / Planning / 評価
- **モデル効率化**: 量子化・蒸留・LoRA・MoE などの実装的進展
- **マルチモーダル / VLM** の新能力・新ベンチ
- **エンタープライズ / 実応用** につながる手法

### 優先度 MEDIUM
- 新しい評価ベンチマーク・データセット
- アライメント / RLHF / DPO 系
- 特定ドメイン（コーディング、数学、医療など）の SOTA

### 優先度 LOW（残り枠で）
- 理論寄りの研究
- 既存手法の小改善

---

## 要約スタイル

FDE職 + AI専門家として「自分・顧客の仕事に組み込めるか」の視点で書く。

- TOP 10 を **エンゲージメント × 重要度** の総合判断で選ぶ
- 各論文に「何が新しいか（技術的コア）」「実務での活用ポイント」を書く
- タイトルは日本語に意訳（直訳不要）
- Bluesky のコミュニティ反応（いいね数・リポスト数）も注目度の指標として簡潔に触れる

## 出力フォーマット

### `note_draft.md`（Obsidianノート本文）

```
---
date: YYYY-MM-DD
source: owlclaw-bluesky
tags: [ai-digest, bluesky, papers, llm]
---

# 🦋 Bluesky 論文ダイジェスト — YYYY-MM-DD

Bluesky の AI/ML コミュニティで話題になった論文 TOP 10。

## 注目論文

### 1. {日本語タイトル（意訳OK）}
- **論文**: {arxiv/HF/OpenReview URL}
- **紹介者**: @{handle} ({author_display}) — ❤️ {like} 🔁 {repost}
- **元ポスト**: {Bluesky URL}
- **技術コア**: {2〜3文}
- **活用ポイント**: {1〜2文、実務での応用}

### 2. ...（計10件）

---
*Generated by owlclaw-bluesky*
```

### `slack.txt`（mrkdwn形式・TOP 5 のみ）

```
:butterfly: *Bluesky 論文ダイジェスト — YYYY-MM-DD*

:scroll: *{日本語タイトル}*
{技術コア1文} | @{handle} ❤️{like}
{論文URL}

...（最大5件繰り返し）

:memo: 残り5件と詳細は Obsidian ノート参照
```

---

## 注意

- 該当論文が0件の場合は `note_draft.md` を作成せず、`slack.txt` には何も書かない（空ファイルもしくは未作成）。
- 5件未満しか選べない場合はその件数で出力してよい（無理に水増ししない）。
- 論文URL が events.md に明記されていればそれを使う。曖昧な場合は元ポストURLのみ記載し「論文URL: 元ポスト参照」と書く。
