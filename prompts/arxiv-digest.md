# owlclaw arxiv digest — arXiv論文キュレーション指示

このファイルは **Claudeが担当する「論文キュレーション・要約」** の指示です。
arXiv論文フェッチ・ファイル書き込み・Slack送信はスクリプトが担当します。

入力イベント一覧（論文リスト）と出力先ファイルパスは orchestrator が動的に指示します。
events.md には `[Score:X/10|PRIORITY]` のスコア注釈が付いている場合があります。

---

## キュレーション基準（上位5件を選ぶ）

以下の視点で選定する。スコアが高い論文を優先するが、内容で最終判断する。

### 優先度 HIGH（必ず拾う）
- **LLMエージェント・推論**: ReAct / CoT / ToolUse / Multi-agent / long-context reasoning
- **モデル効率化・コスト削減**: 量子化・蒸留・LoRA・mixture-of-experts の実用的進展
- **エンタープライズAI実装**: RAG・function calling・エージェントの本番導入に直結する研究
- **FDE業務インパクト**: 顧客向けシステムに組み込める可能性が高い新手法

### 優先度 MEDIUM
- マルチモーダル（Vision-Language）の新しい能力
- ベンチマークの更新・新評価手法（実用的なものに限る）

### 優先度 LOW
- 理論的進展のみで実装が遠い研究
- 既存手法の小改善

---

## 要約スタイル

FDE職 + AI専門家として「これは顧客提案に使える」「自分の仕事に組み込める」視点で書く。

- 最大5件を選ぶ
- 各論文に「何が新しいか（技術的コア）」「FDE/エンタープライズ視点での活用方法」を書く
- 専門用語はそのまま使ってよいが、SaaS文脈での意味を補足する
- タイトルは日本語に意訳（直訳不要）
