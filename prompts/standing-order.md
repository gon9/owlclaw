# owlclaw 共通 Standing Order

全タスクに適用される共通ルール。各タスク固有のプロンプトより優先度は低い（タスク側に矛盾する記述があればタスク側を優先）。

---

## 通知品質ルール

- **ノイズを出さない**: 新着がない・有益な情報がない場合は通知を出さず、ファイルを空のままにする
- **重複しない**: 同一トピックは 1 件にまとめる。複数ソースが同じニュースを扱っていれば、最も詳しいものを 1 件採用
- **ハルシネーション禁止**: 事実確認できない情報は書かない。ソース URL が存在しない情報は「確認できず」と明示するか省略する
- **静かに動く**: 正常完了時の余分な出力は不要。エラー時だけ Slack に 1 行で報告する

## 出力形式ルール

- 日本語で書く。固有名詞（企業名・人名・製品名）は原語を残す
- Slack メッセージは mrkdwn 形式（`*bold*`, `:emoji:` が使える）
- Obsidian ノートは YAML frontmatter を先頭に入れる（フォーマットは各タスクのプロンプト参照）
- URL は省略せず完全な形で記載する

## ファイル操作ルール

- Write するファイルは `tmp/<task-id>/` 配下のみ
- 既存ファイルを Read する場合は絶対パスで指定する
- 書き込み後に確認のための Read は不要（無駄なツール呼び出しを避ける）

## セキュリティルール

- `secrets/` ディレクトリには触れない
- 個人情報（メールアドレス・電話番号・住所）はファイルに書き残さない
- state ファイルに保存するのは ID・タイムスタンプ・数値のみ
